# 🌱 Frontend Practice Tasks — HTML & CSS Clones

A small collection of beginner-level front-end practice projects, built to sharpen my **HTML & CSS** fundamentals by recreating real-world UI designs from scratch.

Each task focuses on a different layout challenge — from a marketing landing page, to a photo gallery, to a full mobile app UI (Instagram) split across three screens.

---

## 🛠 Technologies Used

- **HTML5** — page structure and semantic markup
- **CSS3** — Flexbox, Grid, media queries, gradients, sticky/fixed positioning
- **Font Awesome** (via CDN) — icons
- No frameworks, no JavaScript — pure HTML & CSS practice

---

## 📋 Task 1 — Catchhub (App Landing Page)

A single-page marketing/landing page clone for a fictional app called **"Catchhub"** — built with a self-contained `index.html` file using inline `<style>` CSS.

**What it includes:**
- Hero section with logo, tagline, and app store buttons
- Alternating text/image sections (Channels, Messages, "In Future") using a repeatable `.section` layout
- Centered text blocks and responsive image scaling
- A simple footer with download buttons and a contact link

**What I practiced:**
- Structuring a landing page into reusable `.section` blocks
- Using Flexbox to alternate text and image position across sections
- Basic responsive image sizing with `max-width`

> 📝 Note: This file uses placeholder image paths (`logo.png`, `for_screens.png`, etc.) — replace them with your own image assets before running.

---

## 📋 Task 2 — Catalog-Z Clone (Photo Gallery Template)

A clone of a simple photo/video gallery template called **"Catalog-Z"**, featuring a navbar, hero search bar, image gallery grid, and pagination controls.

**What it includes:**
- Fixed navbar with logo and nav links
- Hero section with a search bar
- Photo gallery grid with date and view-count overlays
- Pagination controls (Previous / page numbers / Next)
- Multi-column footer with links and social icons

**What I practiced:**
- Building a gallery grid layout
- Structuring pagination UI
- Organizing a multi-column footer

> 📝 Note: This file links to an external `style.css` — make sure to create/add that stylesheet in the same folder, and replace the placeholder image paths (`clock.jpg`, `plant.jpg`, etc.) with real images.

## 📋 Task 3 — Instagram UI Clone (3 Screens)

A mobile-first clone of core Instagram screens, split into three separate HTML + CSS pairs.

### 📱 Screen 1 — Profile (`html1.html` + `css1.css`)
- Profile header with lock icon, username, and menu icons
- Profile picture, post/follower/following stats
- Bio section with a GitHub link
- Edit Profile / Share Profile / Add Friend buttons
- Horizontally scrollable "Story Highlights" row
- Post/Reels/Tagged tab switcher
- 3-column photo grid

### 🏠 Screen 2 — Home Feed (`html2.html` + `css2.css`)
- Header with logo, heart, and DM icons
- Horizontally scrollable Stories row with gradient story rings
- A single feed post: profile info, post image, like/comment/share/save icons, likes count, caption, and comments preview

### 🔍 Screen 3 — Explore (`html3.html` + `css3.css`)
- Search bar header
- Responsive 3-column explore grid of images with hover effect

**What all three screens share:**
- A shared bottom navigation bar (Home, Search, Add Post, Reels, Profile) using Font Awesome icons
- Mobile-first layout capped at `max-width: 390px` to mimic a real phone screen
- Sticky headers and fixed bottom navigation

**What I practiced:**
- Recreating a real, well-known app UI pixel-by-pixel
- Horizontally scrollable rows (Stories / Highlights) with hidden scrollbars
- Circular avatars with gradient "story ring" borders using layered backgrounds
- CSS Grid for image galleries with `aspect-ratio: 1`
- Consistent mobile-app-style structure across multiple linked screens

## 🚀 How to Run

No build tools or installation needed — these are plain HTML/CSS files.

1. Download or clone this repository
2. Open any task's `index.html` (or `html1.html`, `html2.html`, `html3.html`) directly in your browser
3. For the best experience with the Instagram screens, view them at a mobile-sized browser width (or use your browser's device toolbar / responsive mode)
